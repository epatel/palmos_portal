# 2048 for PalmOS — Design Spec

## Overview

A classic 2048 sliding tile puzzle game for the Handspring Visor / PalmOS. Pen swipe input, monochrome rendering with inverted tiles for high values, no menus, simple alert on game over, no state persistence.

## Game Rules

Standard 2048:
- 4x4 grid of numbered tiles
- Swipe to slide all tiles in one direction
- Matching adjacent tiles merge (values double, added to score)
- Each tile merges at most once per move
- After each valid move, a new tile spawns: 2 (90%) or 4 (10%)
- Game over when no moves remain (no empty cells, no adjacent matches)

## Game State

```
static int board[4][4];   // 0=empty, 2/4/8/.../2048
static long score;
```

Random seed from `TimGetTicks()` at launch.

## Input

Pen swipe detection:
- `penDownEvent`: record start coordinates (penX, penY)
- `penUpEvent`: compute delta (dx, dy)
- `abs(dx) > abs(dy)` and `abs(dx) > 10`: horizontal swipe (left if dx < 0, right if dx > 0)
- `abs(dy) > abs(dx)` and `abs(dy) > 10`: vertical swipe (up if dy < 0, down if dy > 0)
- 10-pixel minimum threshold prevents accidental taps

## Slide & Merge

For each swipe direction, process each row/column independently:

1. **Compact** — remove zeros, shift tiles toward swipe direction
2. **Merge** — scan in swipe direction: adjacent matching tiles combine (double value, add to score), second tile becomes empty
3. **Compact again** — fill gaps after merging

Each tile merges at most once per move. If board unchanged after processing, no new tile spawns.

**Game over check:** after each move, scan for empty cells or adjacent matching pairs. If neither exists, trigger game over.

## Rendering

**Screen layout** (160x160 pixels):

| Region | Y range | Content |
|--------|---------|---------|
| Title bar | 0-13 | "2048" (PalmOS automatic) |
| Score line | 16-28 | "Score: XXXX" via WinDrawChars |
| Grid | 30-149 | 4x4 tiles, each 36x30, 2px gaps, starting at x=4 |

**Tile rendering:**

| Tile value | Style |
|------------|-------|
| Empty | WinEraseRectangle + WinDrawRectangleFrame (border only) |
| 2-64 | White bg + black border + black number centered |
| 128-2048 | Filled black rectangle + white number (via WinInvertRectangle over text) |

**Number centering:** `FntCharsWidth` to get text width, offset x by `(cellWidth - textWidth) / 2`. Smaller font for 4-digit numbers (1024, 2048).

**Redraw strategy:** redraw entire board only after a valid swipe (turn-based, not continuous).

## App Structure

**Entry point:** `PilotMain` handles `sysAppLaunchCmdNormalLaunch`.

**Startup:**
1. Seed random from `TimGetTicks()`
2. Initialize board to zeros
3. Spawn two random tiles
4. `FrmInitForm` / `FrmSetActiveForm` / `FrmDrawForm`
5. Draw initial board
6. Enter event loop

**Event loop:**
- `EvtGetEvent(&event, evtWaitForever)` — no timeout (turn-based)
- `SysHandleEvent` → `FrmDispatchEvent` for system events
- Custom handler for `penDownEvent` (record coords) and `penUpEvent` (detect swipe, process move)
- `frmOpenEvent`: draw initial board

**Game over:** `FrmAlert` displaying "Game Over! Score: XXXX" with OK button. On dismiss, reset board and start new game.

## What's NOT Included

- No menus
- No state persistence / save game
- No high score persistence
- No animations
- No hardware button input
- No sound
